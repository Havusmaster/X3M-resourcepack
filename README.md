## 🚀 Запуск

Используется `start.bat`, который:
1. Запускает сервер Minecraft в отдельном окне (`java -Xms1G -Xmx1G -jar paper-1.21.11-69.jar nogui`)
2. Открывает монитор ошибок (`server-errors.ps1`), который в реальном времени показывает WARN/ERROR/SEVERE из `logs/latest.log`

Скрипт `server-errors.ps1` фильтрует известные ложные срабатывания (например, ProtocolLib, offline-mode предупреждения).

### 1. **Denizen** (`v1.3.2-b7282-DEV`) — Скриптовый движок ⭐

## 🛠 Для разработчиков

Кастомная логика сервера реализована на **Denizen** — мощном скриптовом движке для Minecraft.
Скрипты находятся в `plugins/Denizen/scripts/`:
- `mechanise/` — системные скрипты (чат, Discord, вход/выход)
- `menu/x3m_menu.dsc` — GUI-меню с кастомизацией
- `custome-item/` — кастомные предметы и оружие

## 💾 Автоматический бэкап мира

Система авто-бэкапа сохраняет все 3 измерения в GitHub каждую ночь в 00:00 МСК.

**Как работает:**
- Скрипт `auto-backup.ps1` запускается вместе с сервером через `start.bat`
- Каждую минуту проверяет время по МСК и количество игроков через RCON
- В 00:00 МСК запускает бэкап:
  - Если есть игроки — ждёт, пока сервер опустеет
  - После пустого сервера ждёт 1 час (чтобы миры точно сохранились)
  - Делает `/save-all`, копирует все 3 мира в `backups/YYYY-MM-DD_HH-MM-SS/`
  - Коммитит в git, пушит на GitHub с тегом `backup-<timestamp>`
  - Удаляет старые бэкапы — хранит только 2 последних
- Ручной запуск: `powershell -File auto-backup.ps1 -Now`

**Настройка GitHub:**
```bash
git remote add origin https://github.com/ВАШ_АККАУНТ/ВАШ_РЕПО.git
git push -u origin master
```

**Структура:**
```
backups/
├── 2026-05-28_00-00-01/
│   ├── world/            # super flat (overworld)
│   ├── world_nether/     # super flat_nether
│   └── world_the_end/    # super flat_the_end
└── 2026-05-27_00-00-01/
    └── ...
```
